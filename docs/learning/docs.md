# Documentation 📖


## Getting Started 🚀
PaperBeam helps teams organize structured content for publishing and collaboration across projects of any size.

## Setup Steps 🛠️
1. Create your workspace
2. Invite your team members
3. Add your content pages
4. Configure your layout settings
5. Publish and manage updates

## Workspace Structure

| Component | Purpose |
|-----------|---------|
| Pages | Individual content documents |
| Sections | Grouped sets of related pages |
| Templates | Reusable layouts for new content |
| Settings | Workspace preferences and access |

::: info
Documentation pages are useful for onboarding new users and explaining workflows to your entire team.
:::

::: tip
You can export any documentation page as a PDF directly from the workspace settings panel.
:::

## Best Practices ✅

- Keep page titles short and descriptive
- Use headings to break up long content
- Add internal links between related pages
- Review content before publishing to avoid drafts going live
